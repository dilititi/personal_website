# Phase 4 · 任务制用户复测

> 目标：验证用户能否在没有讲解产品结构的情况下，完成“选择模板 → 填写核心信息 → 页面内快速编辑 → 调整风格 → 审计与导出”。
>
> 本轮不是继续征集审美意见，而是测量任务完成率、耗时、求助次数和错误路径。

## 1. 测试结论要回答什么

1. 新用户是否能在 30 秒内判断第一步应该做什么？
2. Minimal Portfolio、Personal Journal、Gradient Studio 是否被理解为不同的网站结构，而不只是不同配色？
3. 页面内 `Edit` 是否比进入完整 ContentEditor 更直观？
4. StyleEditor 的基础参数、实时预览和高级参数是否形成清晰层级？
5. 用户是否能理解 Audit 的错误、警告与发布含义？
6. 当前流程顺畅后，用户是否仍明确需要 AI 辅助，而不是因为现有入口难用才提出 AI？

## 2. 参与者与设备分配

至少测试 5 人，每人 20–25 分钟。不要让所有人使用同一模板或同一设备。

| 参与者 | 建议画像                         | 设备             | 指定起点          |
| ------ | -------------------------------- | ---------------- | ----------------- |
| P1     | 第一次制作作品集、非技术用户     | 桌面             | Minimal Portfolio |
| P2     | 设计、摄影或影像专业用户         | 桌面             | Personal Journal  |
| P3     | 熟悉网页或代码的进阶用户         | 桌面             | Gradient Studio   |
| P4     | 主要使用手机、很少使用后台编辑器 | 390px 左右移动端 | Minimal Portfolio |
| P5     | 内容较多的博客、摄影或研究型用户 | 移动端或平板     | Personal Journal  |

如参与者画像不同，保留“三种模板均有人测试、至少两人测试移动端”这两个约束。

## 3. 测试环境

优先使用部署后的候选版本；没有候选部署时使用 production preview：

```bash
npm run build
npm run preview -- --host 127.0.0.1 --port 4174
```

每位参与者开始前：

1. 使用新的无痕窗口，或清除该站点的 localStorage。
2. 确认页面从 Landing 顶部载入。
3. 桌面视口建议 1440×900；移动端使用真实手机或 390×844 模拟视口。
4. 经参与者同意后录屏；主持人另记时间、错误路径和原话。
5. 不预先介绍 ContentEditor、StyleEditor、模板或 Audit 的位置。

## 4. 主持规则

- 先读任务，不讲具体点击路径。
- 鼓励参与者边做边说，但不要替他解释界面。
- 卡住 90 秒后才给一级提示；再过 60 秒给二级提示。
- 一级提示只说目标区域，例如“看看页面顶部的编辑入口”。
- 二级提示才说具体入口，例如“打开 Content，然后看 Start”。
- 每次提示都要记录；主持人代替操作即判定该任务“未独立完成”。
- 任务完成后立即询问单题难度：`1 非常容易 – 7 非常困难`。

## 5. 测试任务

### T0 · 首屏理解

向参与者展示首页，不提供操作说明，然后问：

> 这是什么网站？如果要把它变成你的网站，你准备先点哪里？

记录：

- 说出网站用途所需时间。
- 首次点击是否为 Content 或 Style。
- 是否误把页面导航当作编辑入口。
- 对首页最先注意到的三个元素。

完成标准：30 秒内说出“个人网站/作品集模板”，并找到一个编辑入口。

### T1 · 选择网站类型和视觉方向

任务文本：

> 请把网站设为分配给你的类型。观察网站结构和首页变化，再选择一个你认为合适的视觉方向。

观察：

- 是否找到 Start 或 Style → Templates。
- 是否理解“结构模板”和“视觉预设”的区别。
- 是否查看右侧实时预览。
- 能否说出当前模板与另外两种模板至少两项结构差异。

完成标准：应用指定结构模板，并能指出 Hero、导航、模块数量/顺序、图片比例或排版中的两项差异。

目标时间：2 分钟内。

### T2 · 填写核心信息

任务文本：

> 将站点姓名、个人简介和肖像图片路径改成测试资料。测试资料如下：
>
> - 姓名：`Lin Test`
> - 简介：`Independent maker documenting ordinary places.`
> - 肖像：`/picture/template-organic-portrait.svg`

观察：

- 是否理解五步路径中的“填写核心信息”。
- 是否能找到 SITE 与 ABOUT。
- 是否知道当前语言和另一语言是分开的。
- 修改后是否主动观察页面变化。

完成标准：三项信息均生效且浏览器草稿成功保存。

目标时间：4 分钟内。

### T3 · 页面内快速编辑

任务文本：

> 不回到完整内容表单，直接在 About 页面把章节标题改为 `Field Notes`，并确认页面立即更新。然后找到进入完整编辑器的方法。

观察：

- 是否发现 section 悬停/触摸后的 `Edit`。
- 是否理解快速编辑只包含常用字段。
- 是否看见图片预览与自动保存提示。
- 是否能从快速编辑进入完整 ContentEditor。

完成标准：通过 Inline Quick Editor 修改标题，并打开对应的完整 About 编辑器。

目标时间：90 秒内。

### T4 · 调整风格并比较结果

任务文本：

> 打开风格编辑器，让页面变得更紧凑、标题层级更强，再调整一种高级表现。整个过程中不要关闭实时预览。

观察：

- 是否理解左侧控制、右侧结果。
- 是否一次只操作一个基础参数组。
- 是否能找到 Design/Space 与高级区。
- 是否理解 Texture、Light、Culture、Depth、Mood 的差异。
- 是否使用 Desktop/Mobile 预览切换。

完成标准：

1. 修改至少两个基础参数。
2. 修改至少一个高级参数。
3. 能描述右侧预览发生的可见变化。

目标时间：3 分钟内。

### T5 · 审计与导出

任务文本：

> 假设现在准备发布。请检查网站是否可以发布，说明 Audit 中错误和警告的区别，然后导出内容与风格文件。

观察：

- 是否找到 Audit。
- 是否能运行完整审计。
- 是否理解错误会阻断发布、警告可继续处理。
- 是否能找到 `data.generated.js` 与 `style.generated.js`。
- 是否误以为浏览器草稿已经永久上线。

完成标准：完成审计，正确解释结果，并触发两种导出。

目标时间：4 分钟内。

### 结束访谈

只在所有任务完成后提问：

1. 哪一步最不确定？为什么？
2. 三种模板中，哪些差异最明显？哪些仍像同一个网站？

## 7. 汇总指标与通过门槛

| 指标                  | 通过门槛                                           |
| --------------------- | -------------------------------------------------- |
| T0 找到编辑入口       | 至少 4/5 人在 30 秒内独立完成                      |
| T1 选择结构与视觉方向 | 至少 4/5 人独立完成；中位用时 ≤ 2 分钟             |
| 模板结构差异可感知    | 至少 4/5 人能说出两项非颜色差异                    |
| T2 核心信息填写       | 至少 4/5 人独立完成；中位用时 ≤ 4 分钟             |
| T3 页面内快速编辑     | 至少 4/5 人独立完成；中位用时 ≤ 90 秒              |
| T4 风格工作台         | 至少 4/5 人完成基础 + 高级调整并描述可见变化       |
| T5 审计和导出         | 5/5 找到 Audit；至少 4/5 正确解释错误与警告        |
| 移动端可用性          | 两位移动端参与者均无横向溢出、遮挡或无法点击的阻塞 |
| 严重故障              | 崩溃、数据丢失、无法关闭编辑器、无法恢复均为 0     |
| 推荐意愿              | `愿意使用`评分中位数 ≥ 7/10                        |

## 8. 问题分级与下一轮决策

### P0 · 立即修复

- 崩溃、数据丢失、错误覆盖内容。
- 任务被同一问题阻断 2 人及以上。
- 移动端出现不可操作、明显横向溢出或关键按钮被遮挡。
- 用户无法理解草稿与发布的区别，存在误发布风险。

### P1 · 下一小版本

- 同一任务有 2 人及以上需要提示。
- 中位用时超过目标 50%。
- 模板只能被描述为“颜色不同”。
- 快速编辑和完整编辑器的职责反复被误解。

### P2 · 证据足够再做

- 单人偏好、纯视觉微调或不影响任务完成的问题。
- 新动画、更多模板、更多字段、组件市场等扩展建议。

### 是否进入 P3 Prompt Builder

只有同时满足以下条件才进入：

1. T0–T5 的核心通过门槛已经满足。
2. 至少 3/5 人在没有主持人诱导的情况下，明确提出需要自动生成或 AI 辅助。
3. 他们能指出 AI 应解决的具体任务，而不是笼统说“加 AI”。
4. 该需求可以继续遵守纯静态、无后端和可导出数据的工程边界。

否则下一轮仍优先修复使用成本，不启动 AI API、iframe、后端代理或组件市场。
